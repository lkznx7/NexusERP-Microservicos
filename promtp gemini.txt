# 6. CLEAN ARCHITECTURE

## DOMAIN

O Domain é propriedade exclusiva do usuário.

Considere o Domain uma caixa preta.

O Domain é somente leitura.

Você NÃO pode:

* modificar entidades;
* adicionar atributos;
* adicionar métodos;
* adicionar imports;
* adicionar anotações;
* adicionar frameworks;
* refatorar o domínio.

Você deve agir como se o Domain não existisse para implementação.

O Domain NÃO conhece:

* Spring
* Spring Security
* JPA
* Hibernate
* RabbitMQ
* Redis
* JWT
* OAuth2

O Domain deve permanecer puro.

---

## APPLICATION

Contém apenas:

Os UseCases dependem exclusivamente de Ports.

* JPA
* RabbitMQ

* JWT
* OAuth2
* Frameworks externos

↓

# 12. REDIS

Redis será utilizado para:

* sessões ativas;
* blacklist de tokens;
* rate limiting;
* cache de permissões;
* tokens temporários.

Redis NÃO substitui PostgreSQL.

---

# 13. TRANSACTIONAL OUTBOX

O projeto utiliza Transactional Outbox.

Objetivos:

* consistência eventual;
* confiabilidade na publicação de eventos;
* idempotência.

Sempre considerar essa decisão arquitetural ao discutir RabbitMQ.
